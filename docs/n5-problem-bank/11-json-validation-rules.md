# JSON 검수 규칙서 (N5 문제은행)

## 📋 검수 실행 명령

```bash
node scripts/validate-n5-batch.mjs <json-file-path>
```

---

## ✅ 검수 기준

### 1️⃣ 구조 검증

#### 배열 형식
```
❌ 단일 객체: { ... }
✅ 배열: [ { ... }, { ... } ]
```

#### 배열 길이
```
- 최소: 1개 항목
- 권장: 5-20개 항목
- 최대: 50개 항목 (성능상)
```

---

### 2️⃣ 필수 필드 검증

#### 모든 문제 (record_type 무관)
| 필드 | 타입 | 규칙 | 예시 |
|------|------|------|------|
| record_type | string | "single" 또는 "passage_set" | "single" |
| level | string | "N5" (고정) | "N5" |
| type | string | "문자·어휘" / "문법" / "독해" | "문자·어휘" |
| sub_type | string | 소분류 (필수, 빈 문자열 불가) | "한자 읽기" |
| question | string | 1자 이상, 1000자 이하 | "新しい本を____ました。" |
| answer | string / number | "1", "2", "3", "4" (숫자만) | "1" |

#### 단일 문제 추가 필드
| 필드 | 타입 | 규칙 | 예시 |
|------|------|------|------|
| id | string | 고유 ID (영문자, 숫자, 언더스코어만) | "vocab_kanji_001" |
| option1-4 | string | 각각 1자 이상, 500자 이하 | "かいました" |
| explanation | string | 1자 이상, 1000자 이하 (생략 가능) | "過去形です" |

#### 지문 문제 추가 필드
| 필드 | 타입 | 규칙 | 예시 |
|------|------|------|------|
| group_key | string | 고유 ID (영문자, 숫자, 언더스코어만) | "reading_short_001" |
| content | string | 1자 이상, 5000자 이하 | "田中さんは..." |
| questions | array | 1개 이상, 10개 이하 | [{ ... }] |
| blank_number | number | 1 이상의 정수 | 1, 2, 3 |

---

### 3️⃣ 필드별 상세 검증

#### `level` 검증
```javascript
✅ 허용: "N5"
❌ 불허: "N4", "n5", "JLPT N5", null, undefined
```

#### `type` 검증
```javascript
✅ 허용: "문자·어휘", "문법", "독해"
❌ 불허: "어휘", "한자", "독해력", null, ""
```

#### `sub_type` 검증 (type별)
```javascript
// 문자·어휘
✅ 허용: "한자 읽기", "표기", "문맥 규정"
❌ 불허: "한자", "표기 법", "", null

// 문법
✅ 허용: "판단", "배열", "문맥"
❌ 불허: "문법", "분석", "", null

// 독해
✅ 허용: "단문독해", "중문독해", "정보검색"
❌ 불허: "독해", "읽기", "", null
```

#### `answer` 검증
```javascript
✅ 허용: 1, 2, 3, 4 (숫자 또는 "1", "2", "3", "4" 문자열)
❌ 불허: "①" "ア" "a" 5 0 "1번" null
```

#### `question` 검증
```javascript
✅ 요건:
  - 1자 이상 1000자 이하
  - 최소 1개의 밑줄 (____)
  - 실제 문제 텍스트

❌ 불허:
  - 빈 문자열
  - 밑줄 없음
  - 1000자 초과
  - "문제입니다" 같은 플레이스홀더
```

#### `option1-4` 검증
```javascript
✅ 요건:
  - 각각 1자 이상 500자 이하
  - 중복 없음 (4개 선택지 모두 다름)
  - 실제 선택지 텍스트

❌ 불허:
  - 빈 문자열
  - 선택지 간 중복
  - 500자 초과
  - 정말 불명확한 선택지
```

#### `explanation` 검증
```javascript
✅ 요건:
  - 생략 가능 (null 또는 "")
  - 생성 시 1자 이상 1000자 이하
  - 명확한 해설

❌ 불허:
  - 1000자 초과
  - 무의미한 텍스트 ("예입니다", "맞습니다" 등)
```

### 4️⃣ 논리적 검증

#### 문제와 정답의 일치성
```javascript
✅ 올바른 예:
{
  "question": "新しい本を____ました。",
  "option1": "かいました",    ← 정답
  "option2": "かえました",
  "option3": "かきました",
  "answer": "1"
}

❌ 오류 예:
{
  "question": "新しい本を____ました。",
  "option1": "かいました",
  "answer": "5"  ← 범위 초과
}
```

#### 선택지의 합리성
```javascript
❌ 불합리한 예:
{
  "question": "敬語: 「来てください」を敬語にすると？",
  "option1": "きてください",
  "option2": "りんご",         ← 관련 없는 선택지
  "option3": "ねこ",           ← 관련 없는 선택지
  "answer": "1"
}
```

#### 지문 일관성
```javascript
❌ 불일치 예:
{
  "group_key": "reading_short_001",
  "content": "田中さんは...",
  "questions": [
    {
      "question": "この話の舞台は？",
      "blank_number": 1
    },
    {
      "question": "田中さんの職業は？",
      "blank_number": 3  ← 빈칸 번호 1 건너뜀
    }
  ]
}
```

### 5️⃣ 문화/언어 검증

#### 문제 텍스트 품질
```javascript
✅ 좋은 예:
"日本の季節の中で、____ は最も美しいと言われています。"

❌ 나쁜 예:
"일본의 계절은 ____ 입니다" (한국어 혼용)
"季節について____です question" (영어 혼용)
```

#### 문법적 정확성
```javascript
✅ 올바른 예:
"毎日、大学へ____、勉強しています。"
- 선택지: "行って" (정답)

❌ 부정확한 예:
"毎日、大学へ____、勉強しています。"
- 선택지: "います" (문법 오류)
```

## 🔍 자동 검수 출력

### 성공 케이스
```
✅ 검증 완료: n5_vocab_kanji_batch_01.json
   - 총 20개 항목
   - 모든 필드 유효
   - 논리적 검증 통과
```

### 실패 케이스
```
❌ 검증 실패: n5_vocab_kanji_batch_01.json

   [항목 1] record_type=single
   ⚠️  경고: option1과 option3이 동일합니다
       option1: "かいました"
       option3: "かいました"

   [항목 5] record_type=passage_set
   ❌ 오류: blank_number가 순차적이지 않습니다
       기대: [1, 2, 3, 4]
       실제: [1, 2, 4, 5]

   [항목 12] record_type=single
   ❌ 오류: answer=5 범위 초과 (1-4만 허용)
```

## 📊 검수 통계

각 검증 통과 후 다음 정보 출력:
```
통과율: 100% (20/20)
- 구조 검증: ✅
- 필드 검증: ✅
- 타입 검증: ✅
- 논리 검증: ✅
- 언어 검증: ✅

권장사항:
- 처럼 보이는 선택지가 1개 있습니다 (영어 혼용)
  → 항목 8 option2 검토 권장
```
