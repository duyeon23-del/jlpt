# 신규 유저 콜드스타트 추천 정책

## 기본 원칙
- 풀이 이력이 없는 신규 유저는 특정 skill_key/major_type에 치우치지 않고, 다양한 유형/난이도의 문제를 섞어서 추천받음
- user_stats가 비어 있으면, 각 유형별 기본 점수(0.1~0.5)를 부여해 추천 pool을 넓게 구성
- 추천 함수(get_recommended_questions_by_skill_v3)는 이 정책을 내장하고 있음

## 정책 분기/실험군 예시
- 랜덤: 모든 유형/난이도에서 무작위로 추천
- 난이도별: 2~4단계 난이도를 고르게 섞어 추천
- 유형별: major_type(문법/독해/어휘 등)별로 균등 분포
- 최근 인기/정답률 높은 문제 우선 노출(추후 실험 가능)

## 실무 적용 가이드
- 콜드스타트 정책은 추천 함수 내 기본값(풀이 이력 없음 → 기본 점수)으로 자동 적용됨
- 실험군 분기 필요 시, context/extra 파라미터로 실험군 태깅 및 로그 기록
- 추천 로그(recommendation_logs)에서 신규 유저 cohort 분석 가능

## 참고
- 정책 변경/실험 시 반드시 recommendation_logs에 실험군 context 기록
- 콜드스타트 정책은 운영/실험 환경에서 분리 관리 권장
